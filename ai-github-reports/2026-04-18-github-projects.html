🧠 什么是 Agent-Native？DeepTutor 凭什么不一样

传统 AI 教育工具本质上是"有问必答"的聊天机器人，每次对话独立、没有记忆、无法追踪学习进度。DeepTutor 从根本上重新思考了这个问题：

  • 传统工具：问 → 答 → 结束。下次提问从零开始，AI 不知道你学到哪里。
  • DeepTutor：上传教材 → AI 建立知识图谱 → 多个智能体协作分析你的问题 → 根据你的学习历史和水平生成个性化解释 → TutorBot 积累你的学习画像 → 下次更了解你。

这种"智能体原生"设计的关键在于：所有功能模块共享同一个上下文（UnifiedContext）和事件总线（StreamBus），你可以在聊天中途无缝升级到深度解题或发起研究，历史对话全程有效。

⚡ 五种交互模式：统一线程，按需升级

DeepTutor 最核心的设计之一：五种模式共享同一对话线程，不需要切换界面，可以随时从简单提问升级到深度分析。

💬 Chat — 基础对话
结合文档 RAG 检索和网络搜索，回答基础问题。支持 PDF、LaTeX、图片、代码等多模态输入。与其他模式共享知识库上下文。
🔍 Deep Solve — 深度解题
针对复杂问题启动多智能体推理流水线：检索相关文档片段 → 调用深度推理模型逐步分析 → 生成带引用来源的详细解答。适合高难度习题和概念深究。
📝 Quiz Generation — 自动出题
基于你上传的教材或指定章节，自动生成选择题、填空题、简答题，并即时批改反馈。题目难度根据你的学习历史自动调整。
🔬 Deep Research — 深度研究
同时检索本地文档、Arxiv 学术数据库、网页，综合多源信息生成结构化研究报告。适合撰写论文综述或深入了解某个研究方向。
📐 Math Animator — 数学动画
将抽象数学公式自动转化为逐步动画演示，支持微积分、线性代数、概率统计等领域。用可视化方式理解复杂数学，而不只是看一堆符号。

🏗️ 双层插件架构：Tools + Capabilities

v1.0.0 的最大架构升级是从单体聊天应用转向双层插件模型，使系统高度可扩展：

🔧 Tools 层（原子能力）
封装 RAG 检索、Web 搜索、Arxiv 查询、代码执行、深度推理等基础能力。每个 Tool 实现 BaseTool 协议，可独立复用。
🎯 Capabilities 层（业务模式）
Chat、Deep Solve、Quiz、Research、Math Animator 五个能力模块，调用 Tools 层完成具体任务，实现 BaseCapability 协议。
🌊 StreamBus（事件总线）
所有模块共享统一事件总线，实现模式间的无缝切换和上下文传递。Deep Solve 和 Deep Research 两条流水线虽各自独立，但共享同一 StreamBus。
🧠 UnifiedContext（统一上下文）
跨所有 Capability 的共享上下文对象，保存对话历史、文档片段引用、学习进度等状态，确保五种模式协作一致。

🤖 TutorBot：你的专属持久化导师

DeepTutor 中最具差异化的功能：TutorBot 是一个基于 ReAct（Reasoning + Acting）框架的个性化智能体，它不只是聊天，而是在每次交互中持续建立你的学习画像：

  • 学习历史追踪：记录你学过什么、卡在哪里、强项和弱项分别是什么
  • 跨会话记忆：即使关闭再重新打开，TutorBot 仍然记得上次的进度和你的学习目标
  • 自适应风格：根据你的水平动态调整解释深度——初学者得到类比和直觉解释，进阶者直接获得推导过程
  • 目标导向规划：你可以设定"下周考试"或"深入理解 Transformer"等目标，TutorBot 自动规划学习路径

🚀 快速上手

支持 Docker 一键部署或本地手动安装（需要 Python 3.10+、Node.js 18+）：

# 方式一：Docker 一键启动（推荐）
git clone https://github.com/HKUDS/DeepTutor
cd DeepTutor
cp .env.example .env # 填入 API Key
docker-compose up -d

# 方式二：CLI 接口（纯终端学习）
pip install deeptutor-cli
deeptutor chat --doc textbook.pdf

# 打开 Web 界面
http://localhost:3000

  • Math Animator 是真正的创新：不只是展示公式，而是生成逐步动画，类似请了个会画动画的数学家解题

1. MarkItDown

⭐ 107k 📝 Python 🕐 微软官方

微软开源文档转换神器：让 LLM 轻松读懂 PDF/Word/PPT，支持 15+ 格式一键转 Markdown

文档处理 RAG LLM 格式转换
📖 项目背景

大语言模型（LLM）正成为信息处理的核心工具，但它们的输入能力受限于纯文本。大量有价值的知识以 PDF、Word、PowerPoint 等格式存在，RAG 系统往往难以直接处理这些复杂文档。

MarkItDown 是微软开源的文档转换工具，专注于将 15+ 种文件格式统一转换为 Markdown，让任何 LLM 都能轻松理解文档的结构和内容。

🔧 核心功能
功能类别支持的格式
Office 文档PowerPoint (.pptx)、Word (.docx)、Excel (.xlsx)
其他文档PDF、HTML、Outlook 消息 (.msg)
图片EXIF 元数据 + OCR 识别（通过 LLM Vision）
音频语音转录（Whisper）
其他ZIP 文件、YouTube 视频、EPUB
💡 技术亮点
  • 完整保留 Markdown 格式（标题层级、列表、表格、链接、图片）
  • 插件系统支持第三方扩展
  • 可选 Azure Document Intelligence 进行高质量解析
  • 内置 OpenAI GPT-4o 支持，可对图片描述、音频转录
  • 按需安装：pip install markitdown[pdf] 或 [all]
🚀 快速使用
# 安装
pip install markitdown[all]

# 命令行使用
markitdown document.pdf > output.md
markitdown presentation.pptx > slides.md

# Python API
from markitdown import MarkItDown
md = MarkItDown()
result = md.convert("document.docx")
print(result.text_content)
🎯 适用场景

构建 RAG 数据管道、批量处理历史文档、为 LLM 预处理 PDF 报告

2. llama-stack

⭐ 8.3k 📝 Python 🕐 Meta 官方

Meta 官方 AI 应用部署栈：一行代码切换任意 LLM，OpenAI API 兼容

Llama API服务器 多模型 RAG
📖 项目背景

2026 年是 LLM 应用爆发年，但开发者面临一个核心痛点：如何在不同模型之间无缝切换？

每个模型提供商的 API 格式不同，切换成本高。OpenAI 的生态最成熟，但闭源；开源模型五花八门，部署方式各异。

llama-stack 是 Meta 官方推出的开源智能体 API 服务器，提供统一的接口层，让你的应用可以在 Llama、GPT、Gemini、Mistral 等任意模型之间切换，无需修改一行代码。

🔧 核心能力

1. OpenAI API 兼容

  • /v1/chat/completions - 聊天补全
  • /v1/completions - 文本补全
  • /v1/embeddings - 向量嵌入

2. Responses API（服务端 Agent 编排）

  • 内置工具调用（Tool Calling）
  • MCP 服务器集成
  • 内置 RAG 文件搜索

3. 向量存储与批处理

  • /v1/vector_stores - 向量数据库
  • /v1/files - 文件管理
  • /v1/batches - 离线批处理
💡 技术架构
  • 可插拔 Provider：支持 Ollama、vLLM、托管服务等多种推理后端
  • 本地开发：使用 Ollama 在本地运行，无需 GPU
  • 生产部署：支持 vLLM 分布式推理
  • 多 SDK 支持：OpenAI SDK、Anthropic SDK、Google GenAI SDK
🚀 快速开始
# 安装
pip install llama-stack

# 一键启动本地服务
llama-stack run local

# 或使用 Ollama 后端
llama-stack run --provider-config.yaml ollama

# OpenAI 兼容方式调用
curl https://localhost:8000/v1/chat/completions \
-H "Content-Type: application/json" \
-d '{"model": "llama-4-scout", "messages": [{"role": "user", "content": "Hello"}]}'
🎯 适用场景

多模型对比实验、开源模型生产部署、快速原型验证

3. Microsoft Agent Framework

⭐ 新兴 📝 Python/C# 🕐 微软官方

微软官方多语言 Agent 框架：Python 和 .NET 都能用，图形化编排企业级 Agent

Agent 多语言 微软 图形化编排
📖 项目背景

企业级 AI Agent 的开发需要考虑：生产稳定性、多语言支持、可视化调试、团队协作。现有框架要么只支持 Python，要么偏向研究而非生产。

Microsoft Agent Framework 是微软官方推出的企业级 AI Agent 开发框架，同时支持 Python 和 .NET，配备图形化编排界面，让企业团队能够协作开发、部署和监控 Agent 应用。

🔧 核心功能
功能说明
多语言 SDKPython / .NET 双语言支持，同一套逻辑可跨平台
图形化编排可视化设计 Agent 工作流，非程序员也能参与
工具集成内置 MCP 支持，快速连接外部工具和服务
记忆管理内置短长期记忆，支持向量检索
监控运维内置 tracing 和 metrics，便于生产监控
安全控制内置输入验证和输出过滤
💡 技术架构
  • 三层架构：Orchestrator（编排层）→ Runtime（运行层）→ Tools/Memory/Planner
  • 统一抽象：Python 和 C# 使用相同的概念模型
  • 开箱即用：内置常用工具和记忆组件
🚀 使用示例
# Python SDK
from agent_framework import Agent, Tool

@Tool
def get_weather(city: str) -> str:
return f"{city} 今天晴朗，25°C"

agent = Agent(
name="助手",
instructions="你是一个有用的助手",
tools=[get_weather]
)

result = await agent.run("北京今天天气怎么样？")
🎯 适用人群

企业开发团队（Python/.NET 混合开发）、传统软件团队（有 .NET 技术栈）、DevOps 团队（需要生产级监控）

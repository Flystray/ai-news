MemPalace/mempalace

⭐ 41.9k Python 🆕 2026-04-05 🔥 本周爆火
"The highest-scoring AI memory system ever benchmarked. And it's free."
项目背景

你用了 6 个月 AI，产生了 1950 万 tokens 的对话记忆——但每次关闭窗口，全部归零。传统解法要么依赖 LLM 摘要（成本高、丢失细节），要么付费订阅 Mem0、Zep（$25-249/月）。MemPalace 用一套完全本地的开源系统解决了这个问题。

项目在 LongMemEval 基准测试中以 96.6% Recall@5（零 API 调用） 创下历史记录，可选开启 Haiku Rerank 后达到 100%（500/500）。

核心创新：记忆宫殿架构

MemPalace 借鉴古希腊「记忆宫殿法」，将 AI 记忆组织成可导航的空间结构：

  • Wings（翼）：每个人或项目独立空间，如 wing_kai
  • Halls（厅）：分类走廊，含 facts / events / discoveries / preferences / advice
  • Rooms（房间）：具体主题，同名 Room 在不同 Wing 间自动创建隧道连接
  • Drawers（抽屉）：原始 verbatim 文件，永不摘要，完整保留每一个字
结构不是装饰性的：Wing + Room 过滤相比全库搜索带来 +34% 检索精度提升（60.9% → 94.8%）

  • 适用场景：个人 AI 助手记忆、代码项目上下文持久化、团队知识库构建

02

ghostwright/phantom

⭐ 1.2k TypeScript 🆕 2026-03-26
"An AI co-worker with its own computer. Self-evolving, persistent memory, MCP server."
项目背景

传统 AI 助手是「一次性」的——关闭窗口，上下文归零；你的工具只在你的机器上跑，分享要截图发 localhost 链接。Phantom 的解法是：给 AI 分配一台专属虚拟机，让它像一个真正的同事一样 7×24 小时在线工作。

核心创新：自我进化引擎

Phantom 最独特的能力是「自我进化」——6 步闭环流水线：

  1. Observe：记录对话中的纠正、偏好和领域事实
  2. Critique：评估当前行为与目标的偏差
  3. Generate：生成行为改进建议
  4. Validate：5 门安全关卡（宪法、回归、大小、偏移、安全性）
  5. Apply：将通过验证的改进写入配置
  6. Merge：压缩零散观察为高维「原则」，防止记忆膨胀
三法官投票机制：使用 Claude Sonnet 作为跨模型评判，只要一位法官反对，进化变更即被阻断，防止自我增强偏见。所有版本可追溯、可回滚。
持久记忆实现
  • 三层 Qdrant 向量存储（短期对话 / 长期事实 / 用户偏好）
  • 本地运行 nomic-embed-text 嵌入模型（Ollama），零外部 API
  • 每次对话自动注入相关记忆上下文：周一提到的事，周三它还记得

  • 团队工程助理：代码库解读、基础设施自动化、工具链构建
  • 非技术人员：Slack 对话指令 → Chrome 插件/落地页自动部署

JackChen-me/open-multi-agent

⭐ 5.6k TypeScript 🆕 2026-03-31
"TypeScript multi-agent framework — one runTeam() call from goal to result."
项目背景

构建多智能体系统，你要么用 LangGraph 手绘复杂的状态机图，要么用 Python 生态的 CrewAI，还要搭一堆基础设施。TypeScript 开发者长期被「降级」到调用单个 LLM API 的水平。open-multi-agent 填补了这个空白。

核心创新：目标驱动的自动编排

只需要三行代码：

const team = new OpenMultiAgent({agents: [researcher, coder, reviewer]});
await team.runTeam("构建一个用户认证 REST API");

框架自动：分解目标 → 建立 DAG 任务图 → 解析依赖 → 并行执行 → 合成输出
三种运行模式
  • runAgent()：单智能体单次提示，最简单入口
  • runTeam()：给目标，框架自动规划执行（核心模式）
  • runTasks()：自定义任务图，明确流水线控制
与主流框架对比
框架语言编排方式依赖数量适合场景

技术亮点
  • 3 个依赖：仅 @anthropic-ai/sdk + openai + zod
  • 多模型混合团队：同一团队可混用 Claude / GPT / Gemini / Ollama 本地模型
  • 工具权限控制：预设 readonly / readwrite / full，支持允许/拒绝列表
  • 结构化输出：Zod 模式验证 JSON，可观测性 onTrace 回调
  • 部署无限制：Express / Next.js / Serverless / CI 管道，任意 Node.js 环境
适用场景
  • 自动化研究+编码+审查流水线
  • 多模型协作的内容生成系统
  • 需要在 Next.js/Node.js 后端直接集成多智能体能力的项目
🔗 github.com/JackChen-me/open-multi-agent  ·  安装：npm install open-multi-agent

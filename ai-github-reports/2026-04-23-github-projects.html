RTK（Rust Token Killer）拦截命令输出，在送达 AI 之前完成四层压缩：过滤噪音 → 聚合相似项 → 截断冗余 → 去重重复行。AI 端完全无感知，token 消耗立降 60-90%。

📌 项目背景

开发者在使用 AI 编码工具（Claude Code、Cursor、Copilot 等）时，每次执行 git status、cargo test、pytest 等命令，AI 收到的都是完整的原始输出——大量无关紧要的日志行、冗余信息、格式符号白白消耗 token。RTK 应运而生：它像"中间人"一样拦截命令输出，在送达 AI 之前完成四层压缩。

🚀 核心功能

  • 智能过滤（Smart Filtering）：自动移除注释、空行、模板代码
  • 聚合分组（Grouping）：按目录聚合文件、按类型聚合错误，让 AI 看到的不是 200 行而是 10 行
  • 智能截断（Truncation）：保留相关上下文，裁掉重复冗余部分
  • 去重压缩（Deduplication）：将重复日志行合并为带计数器的简洁条目
  • 透明重写钩子（Auto-Rewrite Hook）：自动拦截 Bash 命令，将 git status 改写为 rtk git status，AI 端完全无感知，100% 自动采用
  • Token 节省分析（rtk gain）：查看各命令的节省统计，识别未被优化的命令
  • 支持 12 种 AI 编码工具：Claude Code、Copilot、Cursor、Gemini CLI、Codex、Windsurf、Cline 等

⚙️ 技术亮点

  • 极低开销：单文件 Rust 二进制，解析开销 <10ms，对开发体验无感知
  • 零依赖安装：Homebrew 一行安装，或 curl | sh 快速部署
  • 命令覆盖全面：100+ 常用命令内置过滤器（git、cargo、npm、pytest、docker、kubectl 等）
  • 失败安全（tee 模式）：命令失败时自动保存完整原始输出，AI 仍可读取，不遗漏调试信息
  • 多平台支持：Linux/macOS 全功能（hook 重写），Windows（CLAUDE.md 注入模式），WSL 最佳体验
  • 隐私优先：遥测默认关闭，仅收集匿名聚合数据，完全符合 GDPR
